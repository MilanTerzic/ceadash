# Power and Energy Unit Audit

Internal energy calculations now use MWh as the base unit. Display code may scale MWh to GWh or TWh through `src/lib/units.ts`; MW remains reserved for power, capacity, average power, NTC, outages and point-in-time flow/load/generation values.

## Notes

- Display-only problems were found in WB6 chart/table scaling and in `fmtMW()` usage on MWh fields.
- Incorrect calculations were found in Serbia period load/generation/net totals, physical-flow net position, and WB6 import/export/net accumulators.
- Ambiguous API unit metadata was found in `fetchLoadGen`; interval duration is now preserved.
- Existing MW uses for NTC, outages, capacity, peak flow, average flow and hourly charts were left as MW.
- Legacy balance and flow routes now redirect to the consolidated system view, so the same unit rules are enforced in the visible `Markets -> System` page.
